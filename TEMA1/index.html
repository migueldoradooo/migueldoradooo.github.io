Posiciones

Este código HTML5 define la estructura de la página usando cuatro elementos div: dos con la clase cuadro para crear figuras de 200x200px (uno rojo arribaiz movido 50px hacia abajo y otro verde derecha posicionado en el extremo derecho de la ventana), uno con la clase bola que es un círculo grande centrado horizontalmente y dimensionado con respecto a la ventana (vw/vh), y un pie que es una barra amarilla de 50px de altura anclada en la parte inferior de la pantalla; el CSS utiliza el selector universal * para eliminar márgenes y rellenos predeterminados, aplica position: relative para el posicionamiento y centrado preciso de los elementos, y define efectos de transición en :hover para que los tres elementos principales (los dos cuadros y la bola) cambien de color y se eleven y escalen ligeramente de forma animada al pasar el ratón.


Bolas

Este código HTML5 establece la estructura de la página utilizando el modelo de caja de CSS, donde un div superior y otro inferior (Parrafo-superior, Parrafo-inferior) actúan como barras de encabezado y pie con bordes y fondo rojo, mientras que el contenido central se organiza mediante un contenedor flexible (contenedor-interior con display: flex) para alinear horizontalmente tres elementos circulares (Parrafo-lateral-1, -2, -3), a los que se suman dos cajas flotantes (float: left y float: right) para la parte inferior, siendo el objetivo principal del código el posicionamiento de estas cajas y la optimización del contenido para compartir en redes sociales gracias al uso intensivo de etiquetas meta de Open Graph y Twitter Cards en el <head>, las cuales definen el título, la descripción y la imagen de vista previa para plataformas como WhatsApp, Instagram y Twitter.


Calculadora

Este código está diseñado para simular la interfaz de una calculadora llamada "Calculadora Hcode" y utiliza un enfoque avanzado de gráficos vectoriales (SVG), en lugar de etiquetas HTML estándar, para construir toda la estructura visual de la carcasa, la pantalla digital y cada botón; la parte del <body> se compone solo de contenedores (<div class="outer"> y <div class="middle">) cuya única función es centrar perfectamente el gráfico SVG en el centro de la pantalla del navegador, y la porción CSS que no define el estilo del SVG se dedica a un extenso "reset" para normalizar la apariencia de elementos en todos los navegadores, mientras que el estilo de los números y el texto de la pantalla se define con una fuente personalizada llamada 'Digital-7'.


Botones

La página es un índice simple que utiliza la etiqueta <h1> para el título principal y las etiquetas <a> (enlaces) que envuelven elementos <button> para crear botones de navegación. Estos botones dirigen al usuario a diferentes archivos HTML (como listas.html, Multimedia.html, Tablas.html y calendario.html), asumiendo que residen en una subcarpeta llamada porfolio (ej. ./porfolio/listas.html); el estilo CSS es minimalista, eliminando márgenes y rellenos por defecto, estableciendo un fondo gris y centrando el título y los enlaces en la página; no obstante, la característica más notable del código es el extenso conjunto de etiquetas <meta> de Open Graph y Twitter Cards en el <head>, cuyo propósito es asegurar que la página se muestre con un título, descripción e imagen consistentes y atractivos cuando se comparta en redes sociales como Twitter, WhatsApp, Instagram y TikTok.


Etiqueta Meta

El código es una página web de ejemplo y repaso que, aunque es visualmente simple, es técnicamente compleja, pues su <head> está muy completo: incluye metadatos básicos de autor y SEO, un favicon para la pestaña, enlaces a dos hojas de estilo CSS externas, estilos internos en cascada para elementos como p, .estiloHead e #estilos-id, enlaces a una fuente de Google Fonts y etiquetas de Open Graph para optimizar la previsualización del contenido en redes sociales, además de vincular un script de JavaScript externo; por su parte, el <body> contiene un enlace (<a>) al sitio web de un IES, un párrafo con un resumen sobre Git, dos elementos interactivos <details>/<summary> que permiten expandir o contraer las definiciones de SEO y Open Graph, y un marco incrustado (<iframe>) que intenta cargar un recurso externo (un mapa), demostrando en conjunto la estructura básica de una página y el uso de la cascada de estilos.


Mi primer JavaScript

Este código HTML es un conjunto de cuatro ejercicios prácticos que demuestran la manipulación directa del DOM de la página usando código JavaScript incrustado en los atributos onclick de los botones, todos sobre un fondo amarillo definido por el CSS; el Ejercicio 1 cambia el texto de un párrafo, el Ejercicio 2 alterna entre tres imágenes de expresiones faciales diferentes modificando el atributo src, el Ejercicio 3 implementa un interruptor de modo claro/oscuro cambiando los estilos de background-color y color del <body> y alternando la visibilidad de los botones Día y Noche, y finalmente, el Ejercicio 4 utiliza sentencias if/else y las propiedades style.display y hidden para hacer que una imagen de un árbol aparezca o desaparezca, actualizando también el texto del botón en cada interacción.


Archivo de Introduccion a CSS

Este código HTML establece una página de inicio simple cuyo objetivo es enlazar y mostrar archivos multimedia locales (una imagen y un video) y proporcionar enlaces de navegación a otras páginas (SEO.html, calculadora.html), con el estilo controlado por una hoja de estilos externa (main.css); sin embargo, la característica más prominente de la página es el extenso conjunto de etiquetas meta de Open Graph y Twitter Cards en el <head>, las cuales se utilizan repetidamente para asegurar que el contenido se muestre de manera optimizada y coherente con títulos y miniaturas atractivas cuando la URL se comparte en diversas plataformas de redes sociales (WhatsApp, Instagram, TikTok y Twitter).


Pagina principal de una Web

La página utiliza un diseño de estructura semántica (<header>, <nav>, <aside>, <main>), donde el <header> (con estilos display:flex y position: relative) contiene un logo, el título "Miguel Ángel", y un enlace de contacto posicionado absolutamente; el menú de navegación (<nav>) enlaza a varias páginas internas (calendario.html, listas.html, etc.) con estilos de espaciado específicos; el cuerpo principal se divide en una barra lateral <aside> (flotada a la izquierda con float: left), que contiene enlaces a recursos educativos y dos <iframe> (uno para un mapa y otro para un traductor de Google), y el área de contenido principal <main>, que presenta una explicación detallada sobre SEO encapsulada en un elemento interactivo <details>/<summary> para ocultar/mostrar la información; además de estos elementos estructurales y el CSS interno para la maquetación, el <head> incluye metadatos básicos y un conjunto completo de etiquetas Open Graph y Twitter Cards para asegurar que la página se comparta de forma atractiva en plataformas sociales.


Mapa Interactivo de Europa

La página presenta un título central (<h1>) sobre un fondo azul claro y utiliza una etiqueta <img> para mostrar el mapa ibérico, al cual se le añade el atributo usemap="#peninsula" para transformarlo en un mapa de imagen interactivo; el elemento <map name="peninsula"> define las áreas clicables, enlazando formas específicas (círculo y polígonos/rectángulos definidos por coordenadas) a sus respectivas páginas de Wikipedia: un círculo para Mallorca, un polígono para España, y rectángulos para Portugal y Francia, mientras que, en el <head>, la página incluye metadatos básicos y un conjunto exhaustivo de etiquetas Open Graph y Twitter Cards para garantizar una previsualización de contenido optimizada al compartirse en plataformas sociales.


¿Que es SEO?

La página presenta una lección sobre SEO sobre un fondo azul, haciendo un uso educativo de numerosas etiquetas semánticas para el formato de texto y la marcación de contenido, tales como <h1> y <h3> para títulos, <abbr> para las siglas (con una imagen anidada en el título principal), <strong> y <em> para énfasis, <,mark>, <del>, e <ins> para ediciones de texto, <dfn> para definir conceptos clave, <pre><code> para mostrar código de ejemplo, <kbd> y <samp> para simular la interacción usuario-sistema, <var> para variables técnicas, <cite> para citar fuentes, y <ul> con <li> para listar puntos, mientras que la sección <head> es extensa y se centra en metadatos para SEO y la optimización de la compartición en redes sociales, incluyendo un conjunto completo y repetitivo de etiquetas Open Graph y Twitter Cards para controlar la apariencia de la página al ser enlazada en plataformas como WhatsApp, Instagram y TikTok.
